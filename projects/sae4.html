SAE S4 · Génie logiciel & DevOps · 2026

Refonte qualité — Carnet de suivi de stage

Chef de projet technique d'une équipe de 6 développeurs. Conteneurisation Docker, normalisation relationnelle, tests E2E.

Contexte

Projet SAE S4 du BUT Informatique IUT2 Grenoble : auditer et refondre une application existante (gestion des stages pour étudiants), avec pour objectif d'améliorer qualité, performances et maintenabilité — sans ajouter de fonctionnalité métier. L'application initiale était instable : crashs Android, modèle de données non normalisé, environnement VM lourd.

Mon rôle

Chef de projet technique d'une équipe de 6 développeurs. Responsabilités :

  • Conteneurisation Docker complète (PostgreSQL 18, PHP-FPM 8.4, Nginx)
  • Rétroconception et normalisation du schéma relationnel en 3FN
  • Mise en place des tests E2E Cypress sur le back-office Symfony
  • Pilotage Agile via Linear (Kanban, traçabilité Issue-Branche-Commit)

Réalisations techniques

1. Conteneurisation Docker

Avant : VM Debian monolithique, dépendances PHP manquantes, environnement non reproductible.

Après : 3 services orchestrés via docker-compose.yml (PostgreSQL, PHP-FPM, Nginx). Déploiement instantané, indépendant de l'OS hôte.

2. Schéma relationnel en 3FN

Avant : Attributs non atomiques (mots-clés concaténés), rôles stockés en JSON, clés sans contrainte métier.

Après : Décomposition complète — rôles en M:N, mots-clés en M:N, clés composites garantissant l'unicité métier.

3. Tests E2E Cypress

Suite de tests validant les parcours utilisateur du back-office (authentification, CRUD offres, navigation). Couverture des régressions sur les flux critiques.

4. Fiabilisation Android

Correction d'un crash Retrofit (espace dans BASE_URL), gestion du cycle de vie via ViewModels, refonte Material Design.

Résultats mesurables

  • Temps de démarrage dev : 15 min (VM) → 1 min (Docker)
  • Requêtes N+1 : ~30 par page → 3-4 optimisées
  • Stabilité Android : crashs systématiques → résolus
  • Schéma BD : violation 2FN → 3FN complète
